# Prompt: build a reusable playbook

A playbook is a saved workflow Claude runs on command. Anything you do more than once as a founder becomes one: investor updates, board prep, pipeline reviews, competitive teardowns.

---

I want to turn this into a reusable playbook.

Here is how I do [name the task], with an example:
[paste your steps or a worked example]

Save this as a Markdown playbook file inside this venture's playbooks folder, with a clear name and a one-line description of when to trigger it. Then tell me the exact phrase I use to run it.

---

Next time, just say: run the [name] playbook.

---

**Playbooks worth building first:**
- investor-update — pulls metrics from Feedback folders, drafts a concise LP/investor email
- board-prep — surfaces the last 30 days of Outputs and Feedback into a board-ready narrative
- pipeline-review — reads all open deals from Inputs, flags what's stalled, surfaces next actions
- competitive-teardown — reads a competitor's latest moves and maps implications to your ICP and GTM
- hiring-brief — given a role, drafts a scorecard, interview questions, and sourcing strategy
